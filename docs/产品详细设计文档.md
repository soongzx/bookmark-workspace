

## 📄 产品详细设计文档（更新版 v2.3）

### **收藏夹工作区 — 产品详细设计文档 v2.3**

#### 1. 项目概述

**项目背景**：作为一个兴趣广泛的探索者，我的浏览器收藏夹在过去十多年里不断膨胀，累积了数千个链接。为了理清这些知识资产，我将其进行了逐层分类，却意外陷入了"分类陷阱"：为了找到某个特定 URL，往往需要像剥洋葱一样逐层展开 3 到 4 级目录，体验十分割裂。由于始终找不到一款能兼顾深度分类与便捷调用的现成工具，我决定亲手打造一款符合自己直觉的解决方案，这便是本项目的诞生初衷。

**Bookmark Workspace Manager（收藏夹工作区）** 是一款 Chrome 浏览器扩展，将传统书签栏中的顶层文件夹抽象为“工作区（workspace）”，并提供多列树状视图、全局搜索、多主题切换、面板缩放、双面板对比视图以及拖放交互，帮助用户高效浏览和管理大量书签。

**版本**：v2.3（2026-05-03）

#### 2. 目标用户

- 拥有大量书签并希望按项目/类别快速切换的从业者
- 需要同时查看多级目录结构的技术人员
- 偏好现代化、可自定义界面的效率工具用户

#### 3. 核心价值

- 屏蔽“书签栏/其他书签”等无关层级，直接聚焦用户自定义的顶层分类
- 横向多列展开，直观体现文件夹层级关系
- 搜索、缩放、主题切换、双面板对比、拖放加载等提升可访问性与工作效率
- 界面状态自动保存，重启即恢复

#### 5. 功能需求详述

##### 5.1 核心概念：工作区（Workspace）

- **定义**：浏览器“书签栏”节点下的直接子文件夹
- **书签栏识别策略**（优先级从高到低）：
  1. 标准 `id="1"` 的节点（Chrome 默认）
  2. 标题含“书签栏”或“Bookmarks bar”
  3. 第一个非“其他书签/Other bookmarks/mobile”的文件夹
  4. 兜底：第一个有子项的文件夹
- **工作区列表**：从书签栏节点的 `children` 中提取所有文件夹

##### 5.2 页面布局（自上而下）

```
┌─────────────────────────────────────────────────┐
│ 标题栏  [📑 收藏夹工作区 Bookmark Workspace Manager v2.3] [搜索][主题][⿰] │
├─────────────────────────────────────────────────┤
│ 工具栏 (半透明背景)                              │
│  [拖拽提示] [🌐全部] [01办公] [02硬件] ...       │
├─────────────────────────────────────────────────┤
│ 展示区容器 (默认单面板，可切换双面板)              │
│ ┌───────────┬────┬───────────┐                  │
│ │ 面板1     │分隔│ 面板2    │                  │
│ │ (激活)    │ 1px│ (可拖)   │                  │

│ └───────────┴────┴───────────┘                  │
```

##### 5.3 标题栏

- 高度 52px，透明背景，无边框，与页面融合
- 左侧显示工具名称"收藏夹工作区" + 英文名称（小字号） + 动态版本号（读取 `manifest.json`）
- 右侧依次排列：
  - 搜索框（宽度 200px，圆角，带搜索图标和清除按钮）
  - 主题切换（四个圆形按钮，对应四套主题）
  - 功能按钮组：清理缓存（🗑️）、切换双面板（⿰）

##### 5.4 搜索栏

- 整合在标题栏右侧
- 输入实时过滤所有书签（标题、URL、路径），200ms 防抖
- 搜索结果列表替换面板1，显示标题、面包屑路径、URL
- `Escape` 或点击清除按钮退出搜索模式，恢复列视图

##### 5.5 工具栏（筛选区）

- 半透明背景 + 模糊滤镜，圆角 10px
- 拖拽提示：鼠标悬浮工具栏时，提示跟随鼠标位置显示
- 筛选按钮：
  - 自动换行
  - 未选中：简单背景，无阴影
  - 选中：半透明强调色背景 + 强调色边框，文字加粗
  - “全部”按钮默认有边框和浅背景以区分
  - 按钮可拖拽至面板加载内容

##### 5.6 展示区

- **默认单面板**，面板2隐藏；可通过 ⿰ 按钮切换双面板
- **双面板**：面板1:面板2 初始宽度 2:1，可通过分隔条拖动调整（范围 20%~80%）
- **面板激活**：
  - 鼠标移入或点击面板任意位置切换激活状态
  - 激活面板标题栏文字变为主题色，背景为浅色强调
- **面板控制**：
  - 每个面板标题栏右侧有“清空”文字按钮，以及缩放控件 (A- / 百分比 / A+)
- **列布局**：
  - 面板内容区域使用 `flex-wrap: wrap`，列宽 220px × 面板缩放系数
  - 层级线条：列左侧有颜色线条，宽度和透明度随层级递减（4px→1px，透明度 1→0.7），颜色跟随当前主题
- **项目样式**：
  - 文件夹：📁 + 名称 + 子项数量 + 展开箭头 `›`
  - 书签：🔗 + 名称
  - 已展开文件夹在父列表中有浅色背景高亮
- **交互**：
  - 点击文件夹 → 展开/折叠右侧层级
  - 点击书签 → 在新标签页打开链接，并 Toast 提示
  - 支持将筛选按钮拖入任意面板的空白区域，直接加载对应工作区

##### 5.7 四套主题

| 主题名称 | CSS 类 | 背景色 | 强调色 | 层级 RGB | 适用场景 |
|----------|--------|--------|--------|----------|----------|
| 暗夜黑金（默认） | dark-gold | #0A0A0A | #D4AF37 (金) | 212,175,55 | 金融/商务 |
| 烈焰暗红 | dark-crimson | #141414 | #DC2626 (红) | 220,38,38 | 游戏/娱乐 |
| 深海蓝夜 | ocean-blue | #0B1426 | #22D3EE (青) | 34,211,238 | 科技/数据 |
| 暖阳浅金 | warm-light | #FDF8E8 | #B4783C (琥珀) | 180,120,60 | 阅读/生活 |

- 所有主题的筛选按钮、层级线、激活背景、Toast 等均使用各自的主题色
- 主题状态持久化在 `localStorage` 中

##### 5.8 缩放控制

- 每个面板独立缩放，范围 0.6x ~ 1.8x，步长 0.1
- 缩放仅影响列宽、字体大小等，通过 CSS 变量 `--panel-scale` 控制
- 缩放值独立保存

##### 5.9 搜索

- 输入防抖 200ms
- 搜索结果替换面板1，列表形式展示
- 搜索时强制隐藏面板2及分隔条
- 退出搜索时自动恢复之前的视图

##### 5.10 状态持久化与清理

- 持久化内容：当前主题、面板1路径、面板缩放值
- 不保存双面板模式（每次打开均为单面板）和面板2路径
- 提供“清理缓存”按钮，一键清除所有 localStorage 数据并重置视图

##### 5.11 滚动条样式

- 横向滚动条高 4px，纵向滚动条宽 3px
- 滑块颜色跟随主题边框色，track 透明，圆角

#### 6. 关键交互流程

- **工作区切换**：点击/拖入筛选按钮 → `loadWorkspaceIntoPanel()` → 更新面板路径 → `renderPanel()`
- **多列展开/折叠**：点击文件夹项 → `handleFolderClick()` → 修改 `panelPath` → `renderPanel()`
- **搜索**：输入文本 → `performSearch()` → 扁平化数组过滤 → 替换面板1内容
- **缩放**：点击 A+/A- → `setPanelScale()` → 更新 CSS 变量 → 保存
- **主题切换**：点击主题点 → `applyTheme()` → 修改 `body` 属性 → 重新渲染面板以更新边框色
- **双面板与分隔条**：点击 ⿰ → 切换 `dualMode` → 显示/隐藏面板2及分隔条 → 调整 flex 比例；拖动分隔条时改变面板 flex 值

#### 7. 安装与使用说明

1. 将所有文件放入文件夹（如 `bookmark-workspace`）
2. 自行准备 16/32/48/128 尺寸的 PNG 图标放入 `icons/` 目录
3. Chrome 浏览器打开 `chrome://extensions`，开启 **“开发者模式”**
4. 点击 **“加载已解压的扩展程序”**，选择该文件夹
5. 点击工具栏扩展图标，新标签页打开使用

#### 8. 设计决策与注意事项

- **书签栏识别兼容性**：多级优先级匹配，适应各种语言和自定义标题
- **数据实时性**：每次打开新标签页均重新调用 `chrome.bookmarks.getTree()`
- **性能**：搜索数据在初始化时构建一次扁平化数组，后续搜索仅为内存匹配
- **层级视觉**：通过边框宽度和透明度表达层级，避免使用过多颜色
- **默认状态**：无默认内容，等待用户操作，避免冗余感
- **拆分原则**：代码按职责拆分为 3 个 JS 文件，便于维护
- **版本动态化**：版本号从 `manifest.json` 读取，减少硬编码
