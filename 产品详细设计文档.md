# Chrome 书签工作区管理器 — 产品详细设计文档 v2.0

## 1. 项目概述
**Chrome Bookmark Workspace Manager** 是一款 Chrome 浏览器扩展，旨在将传统书签栏中的顶层文件夹抽象为“工作区（workspace）”，并提供多列树状视图、全局搜索、多主题切换、面板缩放、双面板对比视图以及拖放交互，帮助用户高效浏览和管理大量书签。

### 1.1 目标用户
- 拥有大量书签并希望按项目/类别快速切换的从业者
- 需要同时查看多级目录结构的技术人员
- 偏好现代化、可自定义界面的效率工具用户

### 1.2 核心价值
- 屏蔽“书签栏/其他书签”等无关层级，直接聚焦用户自定义的顶层分类
- 横向多列展开，直观体现文件夹层级关系
- 搜索、缩放、主题切换、双面板对比、拖放加载等提升可访问性与工作效率


### 2.1 技术选型
- 纯原生 JavaScript，无框架依赖
- 使用 `chrome.bookmarks` API 读取书签树
- 使用 `chrome.tabs.create` 在新标签页打开书签
- 使用 `localStorage` 持久化主题、缩放值
- 界面采用 Flexbox + CSS 自定义属性实现自适应与主题切换

## 3. 功能需求详述

### 3.1 核心概念：工作区（Workspace）
- **定义**：浏览器“书签栏”节点下的直接子文件夹
- **书签栏识别策略**：
  - 遍历根节点的子节点，取第一个**不是“其他书签”且包含子项的文件夹**作为书签栏
  - 兼容标题如“普通书签”、“书签栏”等自定义名称
- **工作区列表**：从书签栏节点的 `children` 中提取所有**非书签项**（即文件夹），按原始顺序渲染
- **“全部”工作区**：虚拟根节点，包含所有工作区文件夹，方便一键浏览

### 3.2 页面布局（自上而下）
```
┌──────────────────────────────────────┐
│ 标题栏  [搜索框] [主题圆点] [⿰]    │
├──────────────────────────────────────┤
│ 工具栏 (半透明背景)                   │
│  ┌────────────────────────────────┐  │
│  │ 拖拽提示（悬浮）               │  │
│  │ [按钮] [按钮] [按钮] ...      │  │
│  └────────────────────────────────┘  │
├──────────────────────────────────────┤
│ 展示区容器                            │
│ ┌─────────┬──────┬─────────┐        │
│ │ 面板1   │ 分隔 │ 面板2   │        │
│ │（缩放） │      │（缩放） │        │
│ │ 可换行  │      │ 单行    │        │
│ └─────────┴──────┴─────────┘        │
└──────────────────────────────────────┘
```

### 3.3 标题栏
- 高度 52px，透明背景，无边框，与页面融合
- 左侧显示工具名称“Bookmark Workspace”（字号 18px，无图标）
- 右侧依次排列：
  - 搜索框（宽度 200px，圆角，内部有搜索图标和清除按钮）
  - 主题切换（四个圆形按钮：暗红、深蓝、暖黄、紫黑）
  - 双面板切换按钮（图标 ⿰，点击激活双面板模式）

### 3.4 搜索栏
- 位置：整合在标题栏右侧
- 输入实时过滤所有书签（标题、URL、路径），200ms 防抖
- 搜索结果列表替换面板1，显示标题、面包屑路径、URL
- `Escape` 或点击清除按钮退出搜索模式，恢复列视图
- 搜索时强制双面板隐藏，列不换行

### 3.5 筛选区（工具栏）
- 整体采用半透明背景 + 模糊滤镜（`backdrop-filter: blur`），圆角 10px，与按钮形成层次
- 内部包含：
  - **拖拽提示**：鼠标悬浮在工具栏时，在顶部显示“💡 拖拽按钮到展示区即可加载内容”
  - **筛选按钮**：水平排列，自动换行（`flex-wrap: wrap`）
    - “🌐 全部”按钮：使用加深的背景色（`accent-light`），边框为主色，字体加粗
    - 其他工作区按钮：常规样式
    - 每个按钮显示工作区名称 + 后代书签数量计数
    - 互斥选择，选中按钮高亮（背景变为主色，文字白色）
    - 按钮可拖拽（`draggable="true"`），拖拽时透明度降低

### 3.6 展示区（多列树状视图）

#### 3.6.1 面板模式
- **单面板模式**：默认状态，仅显示面板1，面板2隐藏
- **双面板模式**：点击 ⿰ 按钮切换，两侧面板均可见，中间有可拖动的分隔条
- 面板内部分为标题栏（显示面板名称 + 缩放控件）和内容区

#### 3.6.2 列布局与换行策略
- **单面板时**：列自动换行（`flex-wrap: wrap`），充分利用宽度，减少水平滚动条
- **双面板时**：列不换行（`flex-wrap: nowrap`），横向滚动
- 列宽 = `220px * 面板缩放系数`，支持通过面板标题栏的“A- / A+”按钮独立调整缩放（0.6x~1.8x）
- 每个面板的缩放值独立保存到 `localStorage`

#### 3.6.3 层级视觉表示
- 使用统一的主题高亮色，在列的左侧边缘以不同宽度和透明度绘制竖线：
  - 第0级：4px 实色（opacity 1）
  - 第1级：3px 实色（opacity 0.9）
  - 第2级：2px 实色（opacity 0.8）
  - 第3级及更深：1px 实色（opacity 0.7）
- 高亮色随主题变化，具体见色彩方案表

#### 3.6.4 项样式
- 文件夹：📁 图标 + 名称 + 子项数量角标 + 展开箭头 `›`
- 书签：🔗 图标 + 名称
- 当前已展开的文件夹在父列表中显示浅色背景高亮，同时左侧显示主色线条

#### 3.6.5 交互
- 点击文件夹 → 展开/折叠右侧层级
- 点击书签 → 在新标签页打开链接，并提示“已打开” Toast
- 支持将筛选按钮拖入任意面板的空白区域，直接加载对应工作区

#### 3.6.6 分隔条
- 仅双面板模式下可见，宽 6px，透明背景，中间有 2px 透明竖线，hover 时显示灰色虚线提示可拖拽
- 支持左右拖动调节两个面板的宽度比例（20%~80%）

### 3.7 四套主题

| 主题名称 | CSS 类 | 背景色 | 卡片色 | 文字色 | 高亮主色 (accent) | 高亮浅色 (accent-light) | 层级颜色 (RGB) | 适用场景 |
|----------|--------|--------|--------|--------|-------------------|------------------------|----------------|----------|
| 暗红（默认） | `dark` | `#1c1c1e` | `#2c2c2e` | `#f5f5f7` | `#ef4444` (红) | `#4c1d1d` | `239,68,68` | 低光环境 |
| 深蓝 | `deep-blue` | `#0f172a` | `#1e293b` | `#e2e8f0` | `#6366f1` (靛蓝) | `#1e1b4b` | `99,102,241` | 夜间专注 |
| 暖黄 | `light-yellow` | `#faf7ec` | `#fffdf4` | `#4d4328` | `#ea580c` (橙) | `#fef3c7` | `234,88,12` | 护眼日间 |
| 紫黑 | `deep-violet` | `#18181b` | `#27272a` | `#f4f4f5` | `#a855f7` (紫) | `#3b0764` | `168,85,247` | 个性化暗色 |

- 所有主题的筛选按钮、层级线、展开高亮背景、Toast 背景均使用各自的主色。
- 主题状态持久化在 `localStorage` 中。

### 3.8 缩放控制
- 每个面板标题栏的右侧有 `A-`、百分比显示、`A+` 三个控件
- 缩放范围 0.6x ~ 1.8x，步长 0.1
- 仅影响本面板的列宽、字体大小、间距等（通过 CSS 变量 `--panel-scale` 实现）
- 缩放值独立保存

### 3.9 全局搜索
- 输入防抖 200ms
- 搜索结果替换面板1内容，不再显示列视图，改为列表形式
- 搜索时强制隐藏面板2和分隔条，列不换行，避免布局混乱
- 退出搜索时自动恢复之前的视图和面板状态

### 3.10 错误与空状态处理
- 加载书签中显示“⏳ 加载书签中...”
- 无法识别书签栏时显示“❌ 未找到书签栏，请检查浏览器书签结构”
- 异常情况显示错误信息
- 空文件夹或空工作区显示“📭 暂无内容”

## 4. 样式设计规范

### 4.1 通用 CSS 变量
所有可配置的样式均通过 CSS 自定义属性定义在 `:root` 和各主题的 `body[data-theme="..."]` 中，实现一键切换。

### 4.2 字体与间距
- 基础字体栈：`-apple-system, BlinkMacSystemFont, "SF Pro Display", "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif`
- 基础字号：13px，随面板缩放动态调整
- 圆角规范：卡片 12px，工具栏 10px，按钮 8px，搜索框 8px
- 阴影：工具栏、面板使用轻微阴影提升层次感

### 4.3 自适应布局
- 页面使用 `flex-direction: column` 垂直布局，`min-height: 100vh`，允许内容撑开高度
- 展示区面板容器使用 `flex: 1 0 auto` 自适应剩余空间
- 筛选按钮区 `flex-wrap: wrap` 自动换行
- 面板内容区域：单面板时 `flex-wrap: wrap`，双面板时 `nowrap`

## 5. 核心交互流程图

### 5.1 工作区切换
```
用户点击/拖入筛选按钮 → loadWorkspaceIntoPanel()
→ 构造新路径 → 更新对应面板的 path 变量
→ renderPanel() → 遍历路径创建列 → 显示内容
```

### 5.2 多列展开/折叠
```
点击文件夹项 → 判断是否已展开：
  展开 → 截断列路径到当前列
  折叠 → 追加新列到路径尾部
→ renderPanel() 更新视图
```

### 5.3 全局搜索
```
输入文本（防抖）→ performSearch()
→ 扁平化书签数组过滤 → 截取前 80 条 → 替换面板1内容为列表
→ 点击结果打开新标签页
→ 按 Escape 或清空搜索 → exitSearchMode() 恢复视图
```

### 5.4 缩放
```
点击 A+/A- → setPanelScale(panelId, value)
→ 更新 CSS 变量 --panel-scale → 更新百分比显示 → 存储到 localStorage
```

### 5.5 主题切换
```
点击主题圆点 → applyTheme(name)
→ 修改 body 的 data-theme 属性 → CSS 变量自动生效 → 存储到 localStorage
```

### 5.6 双面板与分隔条
```
点击 ⿰ → 切换 dualMode
→ 显示/隐藏面板2、分隔条
→ 设置 --panel-wrap 为 nowrap 或 wrap
→ 拖动分隔条鼠标事件更新面板 flex 比例
```

## 7. 安装与使用说明
1. 将所有文件放入同名文件夹（如 `bookmark-workspace`）
2. 打开 Chrome，访问 `chrome://extensions`
3. 开启右上角 **“开发者模式”**
4. 点击 **“加载已解压的扩展程序”**，选择该文件夹
5. 点击浏览器工具栏的扩展图标 → 自动在新标签页打开工作区
6. **图标需自行补充**：从 [iconfont](https://www.iconfont.cn/) 下载四个尺寸的 PNG，放入 `icons/` 目录

## 8. 设计决策与注意事项
- **书签栏识别兼容性**：采用非“其他书签”的第一个文件夹作为书签栏，适应多语言环境。
- **数据实时性**：每次打开新标签页重新调用 `getTree()`，确保数据最新。
- **持久化**：主题、面板缩放分别保存在 `localStorage` 中，恢复用户偏好。
- **性能**：搜索扁平化数组在初始化时构建，搜索仅做内存匹配，无需重复遍历树。
- **层级视觉**：采用宽度和透明度双变量控制，不引入多余颜色，保持界面专业克制。
- **换行策略**：单面板允许列换行，双面板不允许，分别满足空间利用和阅读连续性。
- **搜索模式**：自动隐藏双面板并禁用换行，简化视觉。
