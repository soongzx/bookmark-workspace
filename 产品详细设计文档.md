# Chrome 书签工作区管理器 — 产品详细设计文档

## 1. 项目概述
**Chrome Bookmark Workspace Manager** 是一款 Chrome 浏览器扩展，旨在将传统书签栏中的顶层文件夹抽象为“工作区（workspace）”，并提供多列树状视图、全局搜索、主题切换与缩放控制，帮助用户高效浏览和管理大量书签。

### 1.1 目标用户
- 拥有大量书签并希望按项目/类别快速切换的从业者
- 需要同时查看多级目录结构的技术人员
- 偏好现代化、可自定义界面的效率工具用户

### 1.2 核心价值
- 屏蔽“书签栏/其他书签”等无关层级，直接聚焦用户自定义的顶层分类
- 横向多列展开，直观体现文件夹层级关系
- 搜索、缩放、主题切换提升可访问性与视觉舒适度

## 2. 总体架构与文件结构
```
bookmark-workspace/
├── manifest.json          # 扩展声明
├── background.js          # 点击图标打开新标签页
├── popup.html             # 主界面
├── popup.js               # 全部业务逻辑
└── icons/                 # 图标文件夹（需自行准备）
    ├── icon16.png
    ├── icon32.png
    ├── icon48.png
    └── icon128.png
```

### 2.1 manifest.json 配置要点
```json
{
    "manifest_version": 3,
    "name": "Bookmark Workspace Manager",
    "version": "1.2.0",
    "permissions": ["bookmarks", "storage"],
    "background": { "service_worker": "background.js" },
    "action": { "default_title": "📑 书签工作区" },
    "icons": {...}
}
```
- 权限 `bookmarks` 用于读取书签树
- 权限 `storage` 用于保存主题、缩放偏好（通过 `localStorage` 实现，亦可用 `chrome.storage.local`）

## 3. 功能需求详述

### 3.1 核心概念：工作区（Workspace）
- **定义**：浏览器“书签栏”节点下的直接子文件夹
- **书签栏识别策略**：
  - 遍历根节点的子节点，取第一个**不是“其他书签”且包含子项的文件夹**作为书签栏
  - 兼容标题如“普通书签”、“书签栏”等自定义名称
- **工作区列表**：从书签栏节点的 `children` 中提取所有**非书签项**（即文件夹），按原始顺序渲染
- **全部工作区**：虚拟根节点，包含所有工作区文件夹，方便一键浏览

### 3.2 页面布局（自上而下）
```
┌─────────────────────────────────────┐
│ 标题栏        [主题切换圆点]        │
├─────────────────────────────────────┤
│ 工具栏卡片                           │
│  ┌────────────────────────────────┐ │
│  │ [搜索框]         共 N 个工作区 │ │
│  ├────────────────────────────────┤ │
│  │ [筛选按钮] [筛选按钮] ...    │ │
│  └────────────────────────────────┘ │
├─────────────────────────────────────┤
│ 展示区容器                           │
│  ┌────────────────────────────────┐ │
│  │        [🔍−] 100% [🔍+]       │ │
│  ├────────────────────────────────┤ │
│  │  列1    列2    列3    ...      │ │
│  └────────────────────────────────┘ │
└─────────────────────────────────────┘
```

### 3.3 标题栏
- 固定高度 48px，背景色与下方区分
- 左侧显示“📑 Bookmark Workspace”文字
- 右侧放置**主题切换按钮组**（三个圆形色块）

### 3.4 搜索栏与筛选区（整合为工具栏卡片）
- **搜索栏**：
  - 搜索图标 + 输入框 + 清除按钮
  - 输入实时过滤所有书签（标题、URL、路径），200ms 防抖
  - 搜索结果列表替换展示区，显示标题、面包屑路径、URL
  - `Escape` 或点击清除按钮退出搜索模式，恢复列视图
- **筛选按钮区**（水平滚动）：
  - 动态生成：`全部` + 各工作区按钮
  - 每个按钮显示工作区名称 + 后代书签数量
  - 互斥选择，选中按钮高亮，点击切换对应工作区视图
- 工具栏整体采用圆角卡片样式，内边距 16px 20px，与展示区通过 margin 分隔

### 3.5 展示区（多列树状视图）
- **缩放控件**：展示区右上角 `🔍−` / `🔍+` 按钮，显示当前缩放百分比
- **列布局**：
  - 横向滚动容器，列宽 = `220px * 缩放系数`
  - 每列代表一级文件夹
  - 点击文件夹项 → 在其右侧新增一列，展开其子内容
  - 点击已展开文件夹 → 折叠其右侧所有列（保留到当前列）
- **层级视觉表示**：
  - 列左侧使用**逐渐变细、透明度递增**的彩色边框
  - 规则：
    ```
    第0级（根工作区）：4px 实色（透明度1.0）
    第1级：3px 透明度0.9
    第2级：2px 透明度0.8
    第3级及更深：1px 透明度0.7
    ```
  - 颜色取值由当前主题的 `--level-color` 变量控制（如亮色为 `0, 113, 227`，护眼为 `45, 106, 79`，暗色为 `10, 132, 255`）
- **列内排序**：文件夹在前，书签在后，各自按标题本地化排序
- **项样式**：
  - 文件夹：📁 + 名称 + 子项数量（角标） + 展开箭头 `›`
  - 书签：🔗 + 名称
  - 当前已展开的文件夹在父列表中高亮背景 + 左侧蓝色线条

### 3.6 全局搜索
- 触发方式：聚焦搜索框自动搜索，支持 `Ctrl/Cmd + F` 快速聚焦
- 搜索结果列表替换展示区，最大显示 80 条匹配项
- 每条结果可点击，在新标签页打开对应书签

### 3.7 主题切换
- 三种主题：**亮色**、**护眼**、**暗色**
- 切换方式：点击标题栏右侧三个圆形按钮
- 参数变更：动态修改 `body` 的 `data-theme` 属性，对应 CSS 变量自动生效
- 状态持久化：保存在 `localStorage`

### 3.8 缩放控制
- 缩放范围：0.7x ~ 1.5x，步长 0.1
- 影响范围：列宽、字体大小、间距等所有 `calc(*px * var(--scale))` 计算的值
- 持久化：缩放值保存在 `localStorage`

### 3.9 书签打开
- 点击列视图中的书签项 → `chrome.tabs.create` 新标签页打开
- 点击搜索结果中的书签 → 同上
- 操作后显示 Toast 提示“已打开”

## 4. 样式设计规范

### 4.1 色彩方案（CSS 变量定义）
| 变量名 | 亮色 | 护眼 | 暗色 |
|--------|------|------|------|
| `--bg` | `#f5f6f7` | `#eef5ec` | `#1c1c1e` |
| `--card-bg` | `#ffffff` | `#f9fdf8` | `#2c2c2e` |
| `--text` | `#1d1d1f` | `#2e4a2e` | `#f5f5f7` |
| `--accent` | `#0071e3` | `#2d6a4f` | `#0a84ff` |
| `--level-color` | `0, 113, 227` | `45, 106, 79` | `10, 132, 255` |

所有主题通过 `body[data-theme="..."]` 切换变量组。

### 4.2 字体与间距
- 基础字体：系统原生字体栈（San Francisco / Segoe UI / PingFang SC）
- 基础字号：13px，随缩放动态调整
- 圆角规范：
  - 卡片/工具栏：12px
  - 按钮：8px（筛选按钮）、6px（缩放按钮）
  - 搜索框：10px
- 阴影：低层阴影 `0 1px 3px rgba(0,0,0,0.06)`，温和浮起感

### 4.3 响应式与自适应
- 页面采用 `flex` 垂直布局，高度 `100%`
- 展示区使用 `overflow-x: auto` 实现横向滚动
- 搜索框使用 `min-width` 限制最小宽度，避免挤压

## 5. 核心交互流程图

### 5.1 工作区切换
```
用户点击筛选按钮 → activeWorkspaceId 更新 → switchWorkspace(id)
→ 重置 columnPath → 推入根列数据 → renderColumns()
→ 遍历 columnPath 生成列 DOM → 附加滚动动画
```

### 5.2 多列展开/折叠
```
点击文件夹项 → 判断是否已展开：
  - 若已展开：截断 columnPath 到当前列
  - 若未展开：columnPath 后追加新列
→ re-render 列视图
```

### 5.3 全局搜索
```
输入文本（200ms 防抖）→ performSearch(query)
→ 扁平化书签数组 allBookmarksFlat 中过滤 → 截取前 80 条
→ 渲染搜索列表 → 点击结果打开新标签页
→ 按 Escape 或清空搜索 → exitSearchMode → 恢复列视图
```

### 5.4 缩放
```
点击 + / - → setScale(newScale)
→ 更新 CSS 变量 --scale → 更新百分比显示 → 保存 localStorage
```

## 6. 关键代码实现规范（popup.js）

### 6.1 全局状态管理
| 变量 | 类型 | 说明 |
|------|------|------|
| `workspaceFolders` | Array | 书签栏下的顶层文件夹列表 |
| `activeWorkspaceId` | String | 当前选中的工作区 ID，`__all__` 为全部 |
| `columnPath` | Array | 列视图路径，每元素 `{id, title, children}` |
| `isSearchMode` | Boolean | 是否处于搜索状态 |
| `allBookmarksFlat` | Array | 扁平化所有书签对象，用于搜索 |

### 6.2 关键函数说明
```javascript
// 识别书签栏节点
function findBookmarksBar(rootNode)

// 递归统计子代数量
function countAllDescendants(node)

// 构建层级左边框样式
function getLevelBorderStyle(level)

// 切换工作区并重置列路径
async function switchWorkspace(workspaceId)

// 渲染全部列
function renderColumns()

// 创建单列 DOM
function createColumn(colData, colIndex)

// 创建列内条目（文件夹/书签）
function createItemElement(node, colIndex)

// 文件夹点击折叠/展开逻辑
function handleFolderClick(folderNode, colIndex)

// 搜索入口
function performSearch(query)

// 主题应用
async function applyTheme(theme)

// 缩放控制
function setScale(scale)
```

### 6.3 事件绑定
- 搜索框 `input`、`keydown`、清除按钮 `click`
- `document` 全局快捷键 `Ctrl/Cmd + F` 聚焦搜索
- 缩放按钮 `click`
- 主题切换按钮 `click`

## 7. 安装与使用说明
1. 将所有文件放入同名文件夹（如 `bookmark-workspace`）
2. 打开 Chrome，访问 `chrome://extensions`
3. 开启右上角 **“开发者模式”**
4. 点击 **“加载已解压的扩展程序”**，选择该文件夹
5. 点击浏览器工具栏的扩展图标 → 自动在新标签页打开工作区
6. **图标需自行补充**：从 [iconfont](https://www.iconfont.cn/) 下载四个尺寸的 PNG，放入 `icons/` 目录

## 8. 设计决策与注意事项
- **书签栏识别兼容性**：考虑到不同用户浏览器语言或自定义书签栏标题，采用非“其他书签”的第一个文件夹作为书签栏。
- **数据实时性**：每次打开新标签页时重新调用 `chrome.bookmarks.getTree()`，确保数据最新。
- **主题与缩放持久化**：使用 `localStorage` 存储 `workspace_theme` 和 `workspace_scale`，保证用户偏好不丢失。
- **性能**：全局搜索的扁平化数组在初始化时构建一次，搜索时仅做字符串匹配，无需重复遍历树。
- **层级视觉**：采用宽度和透明度双重变化，避免使用过多颜色，保持界面专业克制；超过3级统一为最细最淡样式，适应大多数实际目录深度。
